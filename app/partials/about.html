Colophon

This site leverages many awesome tools and services that have been built by other developers. Open source libraries and utilities are one of the best things about the web industry in my opinion; it's how we've gotten so far in such a short time.

I started this website by creating a repository on Github for source control, and then scaffolded out the project using Yeoman. That gave me an initial set up including Node and Bower components, to manage dependences like Angular, Gulp, and Autoprefixer, along with a slew of other handy utilities.

Then, I started out with my own custom framework of Sass partials, which allow me to quickly start building out pieces of the site. This framework includes variables, utilities, resets, and base styling for many elements, allowing me to quickly skin a site and write new, custom styles.

I'm a big fan of semantics, performance, and maintainability, which is why I adhere to the principles of Object Oriented patterns in CSS (OOCSS), as well as a BEM naming methodology. This allows me to keep specificity in my CSS classes all on the same level, so I don't go down a path of needing !important statements or overly-specified class names or #id's—or worse, inline styles—in my code. This is hugely important for creating scalable, maintainable code, especially when working with large teams.

The fonts are Freight Micro Pro and Freight Sans Pro, served via Typekit. The icons are a custom set created using Icomoon, so that I can keep a light footprint and only use the icons I need instead of downloading an entire font.
